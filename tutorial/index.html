01 / CHOOSE YOUR INTERFACE

同一个 Codex，四种进入方式。

新手优先选择桌面 App；熟悉终端或编辑器后，再切换到更贴近工作流的入口。

RECOMMENDED PATH

Windows 桌面 App

界面直观，适合把本地文件夹直接交给 Codex。

  1. 01
    安装并登录

    打开 Codex App，使用可用的账号方式完成登录。

    CONNECT
  2. 02
    选择项目文件夹

    打开准备处理的项目，确认工作位置与权限。

    CONTEXT
  3. 03
    发送一个具体任务

    写清目标、已有材料和完成标准。

    RUN
  4. 04
    检查修改结果

    查看变更、运行页面或测试，再决定下一步。

    VERIFY
TERMINAL PATH

Codex CLI

适合熟悉命令行，希望代理直接在当前目录工作的用户。

  1. 01
    打开项目目录

    在终端进入需要处理的仓库或文件夹。

    LOCATE
  2. 02
    启动 Codex

    按当前安装方式启动 CLI 并确认登录状态。

    CONNECT
  3. 03
    描述任务边界

    说明能改什么、不能改什么以及验收命令。

    SCOPE
  4. 04
    审查与运行

    确认差异并执行项目已有的检查命令。

    CHECK
EDITOR PATH

IDE 扩展

适合边写边问，让 Codex 结合编辑器中的文件和选择内容工作。

  1. 01
    安装扩展

    在支持的编辑器中安装并完成授权。

    INSTALL
  2. 02
    打开代码库

    让工作区、当前文件与规则成为上下文。

    OPEN
  3. 03
    从小改动开始

    先处理一个组件、一个报错或一个测试缺口。

    EDIT
  4. 04
    逐步扩大范围

    确认输出可靠后，再交给它跨文件任务。

    SCALE
CLOUD PATH

Codex Web

适合连接代码仓库，把可独立执行的任务交给云端代理。

  1. 01
    连接仓库

    选择目标仓库与任务所在分支。

    LINK
  2. 02
    写清任务

    提供目标、相关位置和验证方式。

    BRIEF
  3. 03
    等待代理执行

    让云端环境读取、修改并运行检查。

    RUN
  4. 04
    审查交付

    查看结果与变更，再合并或继续调整。

    REVIEW
